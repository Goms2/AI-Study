## ❓ 7. 면접 예상 질문 (Interview Prep)

AlexNet의 핵심을 관통하는 면접 질문과 답변 가이드입니다.

### Q1. AlexNet이 이전 모델들보다 혁신적이었던 이유 3가지를 설명하세요.
* **답변 포인트:** 1. **활성화 함수:** ReLU를 도입해 학습 속도를 6배 향상시켰습니다.
    2. **과적합 방지:** Dropout과 Data Augmentation을 통해 깊은 망의 고질적 문제를 해결했습니다.
    3. **병렬 연산:** 2개의 GPU를 사용한 병렬 연산을 구현하여 거대한 데이터셋 학습을 가능케 했습니다.

### Q2. ReLU를 사용했을 때 tanh 대비 학습이 왜 빨라지나요?
* **답변 포인트:** `tanh`나 `sigmoid`는 입력값이 크거나 작을 때 기울기가 0에 수렴하는 **기울기 소실(Vanishing Gradient)** 문제가 발생합니다. 반면 `ReLU`는 양수 영역에서 기울기가 1로 일정하게 유지되므로, 역전파(Backpropagation) 시 신호가 죽지 않고 전달되어 학습 속도가 훨씬 빠릅니다.

### Q3. Dropout은 어떻게 과적합을 방지하고, 테스트 시에는 어떻게 동작하나요?
* **답변 포인트:** 학습 시에는 뉴런을 무작위로 꺼서 특정 뉴런 간의 상호 의존성(Co-adaptation)을 줄여 모델의 일반화 성능을 높입니다. **테스트 시에는 모든 뉴런을 활성화**하되, 학습 시 확률($p=0.5$)만큼 꺼진 것을 보상하기 위해 각 출력값에 $0.5$를 곱해줍니다.

### Q4. AlexNet에서 GPU를 2개 사용한 이유와, GPU 간에 언제 통신이 일어나는지 설명하세요.
* **답변 포인트:** 당시 GPU(GTX 580)의 메모리가 3GB로 부족했기 때문에 모델을 반으로 나누어 병렬로 학습했습니다. 평소에는 각 GPU 내에서만 연산하다가, **Conv3, FC6, FC7, FC8** 레이어처럼 모든 특징을 모아야 하는 시점에서만 GPU 간 통신(Cross-GPU communication)이 일어납니다.

### Q5. Max Pooling과 Overlapping Pooling의 차이는 무엇이고, AlexNet은 왜 선택했나요?
* **답변 포인트:** 일반 풀링은 스트라이드($s$)와 창 크기($z$)가 같아($s=z$) 겹치지 않지만, **Overlapping Pooling**은 창 크기가 이동 거리보다 커서($z > s$, 예: $z=3, s=2$) 영역이 겹칩니다. 논문에 따르면 이 방식이 과적합을 조금 더 방지하고 오류율을 낮추는 효과가 있다고 언급되었습니다.
