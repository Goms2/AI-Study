## Q1. ResNet이 해결하려 한 핵심 문제가 무엇인가요?

> 💡 **힌트**: 더 깊은 네트워크가 더 얕은 네트워크보다 학습 오류가 높아지는 현상의 이름과 원인을 생각해보세요. 그리고 이것이 기울기 소실 문제와 어떻게 다른지도 떠올려 보세요.

<details>
<summary>✅ 모범 답안 보기</summary>

<br>

ResNet은 **성능 저하(Degradation) 문제**를 해결하려 했습니다.

층을 깊게 쌓을수록 훈련 오류(training error)가 오히려 높아지는 현상인데, 직관적으로는 이해하기 어렵습니다. 더 깊은 네트워크는 이론적으로 얕은 네트워크가 하는 일을 그대로 따라할 수 있어야 하기 때문입니다.

**기울기 소실(Vanishing Gradient)과의 차이:**

| 구분 | 기울기 소실 | 성능 저하 (Degradation) |
|------|------------|------------------------|
| **원인** | 기울기가 역전파 과정에서 점점 작아져 앞쪽 층이 학습을 못 함 | 최적화(optimization) 자체가 어려워지는 문제 |
| **발생 시점** | 훈련 초기, 깊은 네트워크에서 | 층을 더 쌓을수록 점진적으로 |
| **과적합 여부** | 무관 | 과적합이 아님 — 훈련 오류 자체가 높아짐 |
| **해결책** | Batch Normalization, 가중치 초기화 | **Residual Learning (ResNet)** |

> 📌 핵심 한 줄 요약: 기울기 소실은 **"신호가 약해지는"** 문제, 성능 저하는 **"최적화 자체가 안 되는"** 문제

</details>

## Q2. Residual(잔차) 학습의 핵심 아이디어를 수식 없이 설명해보세요.

> 💡 **힌트**: "정답을 직접 맞추는 것"과 "현재 답에서 얼마나 수정해야 하는가를 맞추는 것" 중 어떤 게 더 쉬울지 생각해보세요. 그리고 그 "수정분"이 0에 가까워지면 어떤 의미인지도 생각해보세요.

<details>
<summary>✅ 모범 답안 보기</summary>

<br>

기존 신경망은 층이 **정답 자체**를 처음부터 학습해야 했습니다.

ResNet은 발상을 바꿨습니다. 층이 정답 전체를 외우는 대신, **"현재 입력값에서 얼마나 수정하면 정답이 되는가(잔차, residual)"** 만 학습하게 합니다. 그리고 원래 입력값은 숏컷 연결로 출력에 그대로 더해줍니다.

**비유로 이해하기:**

> 📝 시험 답안을 처음부터 쓰는 것 vs. 기존 답안에서 **틀린 부분만 빨간펜으로 수정**하는 것
> → 수정이 훨씬 쉽습니다!

**"수정분이 0에 가까워진다"는 의미:**

만약 해당 층이 더 이상 할 일이 없다면, 수정분을 0으로 출력하면 그만입니다. 그러면 `출력 = 0 + 입력 = 입력` 이 되어, **층이 없는 것처럼** 동작할 수 있습니다. 기존 방식에서는 이 "아무것도 안 하기"를 학습하는 것조차 어려웠습니다.

```
기존 방식: 층이 정답 H(x) 를 통째로 학습
ResNet  : 층이 수정분 F(x) = H(x) - x 만 학습
          최종 출력 = F(x) + x  (숏컷 연결로 x를 더해줌)
```

</details>

## Q3. Shortcut connection이 역전파(backpropagation)에서 어떤 이점을 가져오나요?

> 💡 **힌트**: 역전파(backpropagation)는 출력에서 입력 방향으로 기울기를 전달합니다. 숏컷 연결이 있으면 기울기가 어떤 "추가 경로"로 흐를 수 있을지 생각해보세요.

<details>
<summary>✅ 모범 답안 보기</summary>

<br>

숏컷 연결은 역전파 시 **기울기가 흐를 수 있는 추가 경로**를 만들어줍니다.

**일반 네트워크의 역전파:**
```
출력 → 층N → 층N-1 → ... → 층1 → 입력
       (기울기가 층마다 곱셈을 통해 전달 → 점점 작아질 수 있음)
```

**ResNet의 역전파:**
```
출력 → 층N → 층N-1 → ... → 층1 → 입력
  └──────────────────────────────┘  ← 숏컷 경로 (기울기가 "그대로" 통과)
```

**핵심 이점 2가지:**

| 이점 | 설명 |
|------|------|
| **기울기 고속도로** | 숏컷 경로로 기울기가 줄어들지 않고 앞쪽 층까지 직접 전달됨 |
| **기울기 소실 완화** | 어떤 층의 기울기가 아주 작아져도, 숏컷 경로의 기울기(최소 1)가 더해지므로 완전히 사라지지 않음 |

> 📌 숏컷 연결의 덧셈(+) 연산은 역전파 시 기울기를 **그대로 복사**해서 전달하기 때문에, 150층 이상의 네트워크도 안정적으로 학습할 수 있습니다.

</details>

## Q4. Bottleneck Block에서 1×1 conv를 사용하는 이유가 무엇인가요?

> 💡 **힌트**: 1×1 conv는 공간적 정보를 바꾸지 않고 채널 수만 줄이거나 늘릴 수 있습니다. 3×3 conv의 입력 채널을 줄이면 계산량에 어떤 변화가 생기는지 생각해보세요.

<details>
<summary>✅ 모범 답안 보기</summary>

<br>

**1×1 conv의 역할: 채널 수 조절 (계산량 절감)**

3×3 conv는 채널 수가 많을수록 계산량이 폭발적으로 증가합니다. 1×1 conv로 채널을 먼저 줄이면, 3×3 conv가 훨씬 적은 계산량으로 특징을 추출할 수 있습니다.

**Bottleneck 구조의 흐름:**

```
입력 (256채널)
    │
  1×1 conv  →  64채널  (채널 4배 축소 = 계산량 대폭 감소)
    │
  3×3 conv  →  64채널  (핵심 특징 추출, 채널이 적어 계산 효율적)
    │
  1×1 conv  →  256채널 (채널 복원)
    │
출력 (256채널)
```

**계산량 비교 (입출력 256채널, 56×56 feature map 기준):**

| 방식 | 구조 | 연산 수 (대략) |
|------|------|---------------|
| Basic (3×3 × 2) | 256 → 256 → 256 | 약 **1,849M** |
| Bottleneck | 256 → 64 → 64 → 256 | 약 **69M** |

> 📌 **핵심 한 줄**: 1×1 conv는 채널을 줄여 **"연산의 병목"** 을 만들고, 덕분에 훨씬 깊은 네트워크를 같은 계산량으로 쌓을 수 있습니다.

</details>

## Q5. ResNet-50과 ResNet-34의 구조적 차이점은 무엇이고, 왜 다른 블록 설계를 사용하나요?

> 💡 **힌트**: ResNet-34는 Basic Block(3×3 conv 2개), ResNet-50은 Bottleneck Block(1×1 + 3×3 + 1×1 conv 3개)을 사용합니다. 두 블록의 계산량과 표현력 차이를 비교해보세요.

<details>
<summary>✅ 모범 답안 보기</summary>

<br>

**블록 구조 비교:**

| 항목 | ResNet-34 (Basic Block) | ResNet-50 (Bottleneck Block) |
|------|------------------------|------------------------------|
| **구조** | 3×3 conv → 3×3 conv | 1×1 conv → 3×3 conv → 1×1 conv |
| **conv 수** | 2개 | 3개 |
| **채널 변화** | 유지 | 축소 → 처리 → 복원 |
| **파라미터 수** | 21.8M | 25.6M |
| **계산 효율** | 채널이 적을 때 효율적 | 채널이 많을 때 효율적 |

```
Basic Block (ResNet-34)       Bottleneck Block (ResNet-50)

  x                              x
  │                              │
  ├──────────┐                   ├─────────────────┐
  │          │                   │                 │
3×3 conv     │               1×1 conv (축소)       │
3×3 conv     │               3×3 conv (처리)       │
  │          │               1×1 conv (복원)       │
  └──(+)─────┘                   │                 │
      │                          └─────(+)─────────┘
     출력                               │
                                      출력
```

**왜 ResNet-50부터 Bottleneck을 쓰나요?**

ResNet-34까지는 채널 수가 비교적 적어(최대 512) Basic Block으로도 충분히 효율적입니다. 하지만 ResNet-50부터는 층이 더 깊어지고 채널이 많아지면서, Basic Block을 그대로 쓰면 **계산량이 너무 커집니다.**

Bottleneck Block은 1×1 conv로 채널을 줄여 3×3 conv의 계산 부담을 낮추기 때문에, **더 깊은 네트워크를 합리적인 계산 비용으로 만들 수 있습니다.**

> 📌 **한 줄 요약**: Basic Block은 **단순하고 얕은 네트워크**에, Bottleneck Block은 **깊고 표현력 높은 네트워크**에 적합한 설계입니다.

</details>
